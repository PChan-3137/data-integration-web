# 학생 출결 관리 예제 프로젝트
## 파일 구성
`index.html` - 메인 HTML

`syle.css` - 스타일 시트

`script.js` - 내부 처리 로직

`server-data.js` - 외부 서버(가정) 제공 학생 데이터

---

## 1. 외부 시스템과 내부 시스템 구분

### 외부 시스템
본 프로젝트는 `server-data.js` 파일을 외부 학생 관리 서버로 가정했음.

- 학생 출결 정보 관리/보유
- `<script src="server-data.js"></script>` 태그를 통해 파일을 불러옴
- 실제 환경에서 서버 데이터를 받아오는 것을 가정함

---

### 내부 시스템

- `index.html` : 화면 구조 정의
- `style.scss / style.css` : 화면 스타일 정의
- `script.js` : 데이터 처리 및 화면 렌더링

---

## 2. 연계 데이터 식별

| 데이터 항목 | 설명 | 화면 사용 여부 |
|-------------|------|----------------|
| `name` | 학생 이름 | O |
| `className` | 학년/반 정보 | O |
| `status` | 출결 상태 | O |
| `id` | 식별자 | X |

---  

## 3. 데이터 연계 흐름

1. 외부 시스템(`server-data.js`)에서 학생 출결 데이터 제공
2. HTML에서 외부 데이터 파일 로드 (`<script src="server-data.js"></script>`)
3. 내부 JavaScript(`script.js`)에서 `serverStudents` 데이터 접근
4. 초기 로딩 시 전체 학생 목록 화면 출력 (`renderList(serverStudents)`)
5. 사용자가 상태 탭(전체/출석/지각/결석) 클릭
6. 선택된 상태 기준으로 데이터 필터링 
7. 필터링 결과를 기반으로 학생 카드 및 개수 화면 렌더링

```text
외부 서버 데이터
(=server-data.js)
      ↓
  script.js
      ↓
 데이터 필터링
      ↓
   DOM 생성
      ↓
  웹 화면 출력
```

## 4. 본 과제가 데이터 연계 구현 과제인 이유에 대한 설명

- 외부 시스템 역할을 하는 학생 관리 데이터 서버(`server-data.js`)와 내부 시스템(`script.js`)를 구분
- 외부 데이터 파일을 수정하지 않고 그대로 받아와 작동
- 사용자 조작에 따라 외부 데이터가 내부 시스템에 따라 동적으로 처리(`filter`)되고 화면에 반영

독립적으로 기능할 수 있는 두 시스템이 서로 연계되어 하나의 결과물로 완성되기 때문
